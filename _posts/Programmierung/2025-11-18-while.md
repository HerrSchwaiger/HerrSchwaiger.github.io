Manchmal muss ein Code-Abschnitt beliebig oft wiederholt werden. Genau für diesen Zweck gibt es die klassische kopfgesteuerte `while`-Schleife und die fußgesteuerte `do-while`-Schleife. In diesem Beitrag schauen wir uns beide Varianten genau an und lernen, wann welche Form die richtige Wahl ist.

## Die kopfgesteuerte While-Schleife

Die klassische `while`-Schleife prüft die Bedingung am Anfang, also im "Kopf" der Schleife. Erst wenn die Bedingung wahr ist, wird der Schleifenkörper ausgeführt.

### Syntax

```java
while (bedingung) {
    // Code, der wiederholt werden soll
}
```

### Beispiel: Countdown

```java
int countdown = 3;
while (countdown > 0) {
    System.out.println(countdown);
    countdown--;
}
System.out.println("Start!");
```

Ausgabe:
```bash
3
2
1
Start
```

{: .notice--warning}
**Wichtig zu beachten:**\\
Wenn `countdown` von Anfang an 0 oder kleiner wäre, würde die Schleife überhaupt nicht ausgeführt werden.

## Die fußgesteuerte Do-While-Schleife

Die `do-while`-Schleife unterscheidet sich in einem entscheidenden Punkt: Sie prüft die Bedingung erst am Ende, also im "Fuß" der Schleife. Das bedeutet, der Schleifenkörper wird mindestens einmal ausgeführt, unabhängig davon, ob die Bedingung wahr oder falsch ist.

### Syntax

```java
do {
    // Code, der wiederholt werden soll
} while (bedingung);
```

{: .notice--primary}
Beachte das Semikolon am Ende – das wird häufig vergessen!

### Beispiel: Benutzereingabe

Ein klassischer Anwendungsfall für die `do-while`-Schleife ist die Validierung von Benutzereingaben:

```java
import java.util.Scanner;

Scanner scanner = new Scanner(System.in);
int zahl;

do {
    System.out.print("Bitte gib eine Zahl kleiner gleich 10 ein: ");
    zahl = scanner.nextInt();
} while (zahl > 10);

System.out.println("Danke! Du hast " + zahl + " eingegeben.");
```

Hier wird der Benutzer mindestens einmal nach einer Eingabe gefragt. Solange die Eingabe ungültig ist, wird die Schleife wiederholt.

## Der entscheidende Unterschied

Der Hauptunterschied zwischen `while` und `do-while` liegt in der Garantie der Ausführung:

**While-Schleife**: Kann null bis n-mal ausgeführt werden

```java
int x = 10;
while (x < 5) {
    System.out.println("Dies wird nie ausgegeben");
}
```

**Do-While-Schleife**: Wird mindestens einmal ausgeführt

```java
int x = 10;
do {
    System.out.println("Dies wird einmal ausgegeben");
} while (x < 5);
```

## Vorsicht vor Endlosschleifen!

Einer der häufigsten Fehler beim Programmieren mit Schleifen sind unbeabsichtigte Endlosschleifen:

```java
// FEHLER: Endlosschleife!
int i = 0;
while (i < 10) {
    System.out.println(i);
    // i wird nie erhöht!
}
```

Um dies zu vermeiden, stelle sicher, dass die Abbruchbedingung irgendwann erfüllt wird.
